# Contributing to Fiely

Thanks for your interest in Fiely! We're in early design phase and welcome ideas, feedback, and code.

## Ways to contribute

- 💬 **Discuss** — Open an issue to share ideas, ask questions, or suggest features
- 🐛 **Report bugs** — File an issue with clear reproduction steps
- 📖 **Improve docs** — Docs PRs are always welcome
- 🛠️ **Code** — Pick up an open issue or propose a change first in an issue

## Before you start coding

Because Fiely is in early design phase, please **open an issue first** before working on a larger change. This avoids duplicate work and helps us align on direction.

## Development setup

Coming soon — we're still bootstrapping the project. Keep an eye on the README for updates.

## Commit style

- Use clear, descriptive commit messages
- Prefer small, focused commits over large ones
- Reference issues where relevant (e.g. `Fixes #42`)

## Code of Conduct

Be kind. Be respectful. Assume good intent. We're all here because we care about data sovereignty and good software.
